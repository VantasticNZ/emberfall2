# WORLD GRAPH — the content-sized 2D node+edge map (the spec behind the drawing)

> **Design — the fast graph-first world layout (Van's method).** The world is sized **FROM the
> content**: every node is scaled by what the story/quests need there, so it **cannot be too small by
> construction**. The DRAWN map is the key deliverable — **`docs/world-graph.png`** (rendered from
> **`docs/world-graph.svg`**, generated by `scripts/worldgraph/gen-worldgraph.mjs`; the node/edge data
> in that script IS this spec). **DESIGN ONLY — no region/nav building yet** (that's the next stage,
> after Van approves + adjusts the graph). Sized from the `WORLD-SCALE-DOD.md` content audit (66
> quests · 30+ NPCs · 40+ places). Cross-refs: all `src/data/quests/*`, `gating.js`, `STORY-AND-QUESTS`,
> `NPC-WORLD`, `ALIGNMENT-SYSTEM`, `MECHANICS-DESIGN`, `CONNECTORS`, `MASTER-MAP`.

## THE DRAWING — `world-graph.png`
A 2D map of **54 nodes / 55 edges**: nodes are **sized by content load** (a huge city holding many
quests = a big star; a single-quest spot = a small dot), **coloured by region/biome**, and connected
by **paths** — **dashed + 🔒 = tool-gated** (per `gating.js`), **dashed grey = new biome (not yet
gated)**. The **river spine** (blue, N→E delta) and the **biome tints** show the geography. Layout =
the real hub-and-spoke: **Greenhollow centre**, Marsh W, Peaks/Spire N, Coast E, Emberwood S, + the
new **Forest** & **Desert** expansion biomes SE. Edit the script + re-run to adjust.

## HOURS / DENSITY SANITY
**66 quests × ~25 min ≈ 27-33 h** of quest content **+** exploring **3 huge cities** (6-10 streets each)
+ ~8 towns + 5 villages + **14 dungeons/caves** + 9 secrets + the new biomes **+** combat encounters
**+** the 5-ending replay value **= ~30-45 h.** ✅ **The placed content meets the 30+ h target** — and
the graph being sized to it means the world *holds* the game instead of cramming it (~12 h) as the
old too-small map did.
